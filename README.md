# WebRTC звонок 1:1 (GitHub Pages)

Это минимальное web‑приложение для аудио/видео звонка **1‑на‑1** через WebRTC.

Можно в двух режимах:

- **авто‑режим (по юзернейму)**: вводите ник → второй звонит по нику (Offer/Answer/ICE идут автоматически через Supabase Realtime)
- **ручной режим** (запасной): обмен Offer/Answer (или `RS1:...`)

## Запуск локально

Требуется Node.js (у вас уже есть).

```bash
npm run dev
```

Откройте ссылку, которую покажет `serve`.
Важно: на телефоне доступ к микрофону/камере обычно **не работает по http** — нужен **https** (например, GitHub Pages).

## Как позвонить

1. Оба откройте страницу.
2. Оба нажмите “Включить микрофон (и камеру)”.
3. **Авто‑режим**:
   - оба вводят “Ваш юзернейм” и жмут “Войти”
   - один вводит “Кому звонить” и жмёт “Позвонить”

Примечание: приложение старается выдавать **короткий код** в формате `RS1:...` (это тот же Offer/Answer, только сжатый). Если по какой-то причине сжатие недоступно, будет обычный JSON.

## Деплой на GitHub Pages

1. Создайте репозиторий на GitHub и залейте туда файлы `index.html`, `app.js`, `style.css`.
2. В репозитории откройте **Settings → Pages**.
3. В **Build and deployment** выберите:
   - **Source**: `Deploy from a branch`
   - **Branch**: `main` (или `master`)
   - **Folder**: `/ (root)`
4. Сохраните — GitHub даст ссылку на сайт.

## Авто‑режим (Supabase Realtime)

1. Создайте проект на Supabase.
2. В проекте откройте **Project Settings → API** и возьмите:
   - **Project URL**
   - **anon public key**
3. Откройте файл `config.js` в репозитории и вставьте эти значения в:
   - `window.SUPABASE_URL`
   - `window.SUPABASE_ANON_KEY`
4. Закоммитьте и залейте — GitHub Pages подхватит.


## Ограничения

- В некоторых сетях прямое P2P‑соединение может **не установиться** без TURN‑сервера (это ограничение WebRTC/NAT, а не этого проекта).
- Если нужен максимально “железный” коннект во всех сетях, тогда потребуется TURN и, значит, какой‑то хостинг для него.
